
# 📝 Script Standard

  - **概述**: 当**创建一个新的 Python 脚本**（`.py`）时，请按照下面的一些规范进行脚本编写的约束，且注意保持代码简洁美观。
  - **日期**: 2026-03-12

## 1 script-header

  - **在文件的最开始添加以下示例代码块，放在任何导入或代码之前。**

    - **@Author**：写作者或团队名；用户未指定时可用「作者」或留空让用户补。
    - **@Date**：使用**当前日期**，格式 `YYYY-MM-DD`。
    - **@Description**：脚本功能概述，根据脚本用途写一句话说明。
    - **@Command**：若有典型运行方式（如 `python inference.py`、`PYTHONPATH=src bash xxx.sh`）则写；否则可省略整行。

  - **示例结构**

    ```python
    """
    # @Author: AI技术平台
    # @Date: 2026-03-12
    # @Description: Qwen3-VL 推理脚本，支持单张图片与视频
    # @Command: python inference.py
    """
    ```

  - **注意**：仅在对**新建脚本**或用户**明确要求加文件头**时添加；修改已有脚本时不要强行覆盖原有头部。

## 2 Script-code

  - **脚本整体结构**：除了一些特定的函数和类存储脚本外，可执行的新脚本一般遵循基本结构如

    - `功能函数`: 能独立成模块的逻辑单元尽量下沉为独立函数，

    - `主调函数`: 不在主调层写复杂逻辑，仅保持"调用-打印-调用"的线性可读性。
    
    - `__main__`: 执行入口，仅负责参数处理和传递

  - **函数间距**：顶层函数之间保留 **2 个空行**。

  - **代码风格**: 变量易读，简洁高效，逻辑清晰。

  - **示例结构**

    ```python

    import os
    from PIL import Image

    #-------------#
    # 创建输入信息
    #-------------#
    def build_messages(description, file_path):
        ...

    #-------------#
    # 推理函数
    #-------------#
    def inference(weight, file_path, device):
        messages = build_messages(description, file_path)
        ...

    #-------------#
    # 参数(非必须)
    #-------------#
    def parse_args():
        ...

    if __name__ == "__main__":
        args = parse_args()
        inference(...)
    ```

  - **注意**：调用子函数时，除非个别超多参数，需要换行添加注释，否则就写到一行当中，保持一行一个命令的简洁方式；


## 3 Script-annotation

  - **函数内功能块注释**：使用对称短横线框住标题，标题与框线之间留一个空格，长度保持一致。框线长度要尽量大于注释的标题长度。

    ```python
    #-------------#
    # 加载模型
    #-------------#
    ```

  - **行尾注释**：注释内容与 `#` 之间保留 **1 个空格**。相邻代码块的行尾注释应上下行左对齐。示例如下

    ```python
    #----------------------------#
    device = args.device         # GPU 编号，或 "auto"、"cpu"
    #----------------------------#
    weight_path = args.weight    # 模型路径
    file_path = args.filepath    # 文件/文件夹路径
    #----------------------------#
    ```

  - **注释风格**：简洁高效，逻辑清晰。

  - **注意**：1. 可在关键信息处适当采用 emoji 进行打印或代码注释；2. 避免在显而易见的代码行上加注释（如 `files.sort()` 无需注释）。
