
# README 编写规范

适用于**新建或改写项目 README**（如 `README.md`）时的结构与格式规范，目标是简洁易读。

> 仅对**新建 README** 或用户**明确要求按规范写**时应用；修改已有 README 时保留原有风格，只在不冲突处做增强。

---

## 文档头部

- **标题**：首行使用一级标题，可配一个与主题相关的 emoji，后跟项目/文档名称。

  ```markdown
  # 🚀 项目名称 / 使用指南
  ```

- **简介**：标题下加 1～2 句简短说明，不写长段落。

---

## 章节结构

- **层级**：主章节用数字编号（如 `## 1 环境搭建`、`## 2 使用方式`），子节用 `###`（如 `### 3.1 数据集制作`）。
- **分隔**：主章节之间用 `---` 分隔，便于阅读。
- **列表**：要点用 `-`，需要强调的标签用 `**粗体**`；多级列表用 2 空格缩进。

示例结构：

```markdown
## 1 环境搭建
...

---

## 2 使用方式
...

## 3 进阶说明
### 3.1 子章节 A
### 3.2 子章节 B
```

---

## 目录结构图

凡涉及**项目结构**或**数据集目录约定**，统一用无语言标记的代码块 + ASCII 树形图展示，右侧行内注释说明用途，注释内容左对齐（用空格对齐到同一列）：

```
项目名/
├── 目录A/          # 简短说明
│   └── 文件.yaml   # 简短说明
├── 目录B/          # 简短说明
├── 入口文件A.py    # 简短说明
└── 入口文件B.py    # 简短说明
```

```
YOUR_DATA_DIR/
├── train/
│   ├── A           # 时相 A 图像
│   ├── B           # 时相 B 图像（与 A 同名）
│   └── label       # 变化标签
└── val/
    ├── A
    ├── B
    └── label
```

- **无语言标签**：结构图不加 `bash` / `text` 等语言标记，保持简洁。
- **注释对齐**：`#` 注释尽量对齐到同一列，提升可读性；无需注释的行留空即可。
- **只写关键路径**：无需穷举所有文件，聚焦核心目录与入口。

---

## 代码与命令

- **代码块**：必须标明语言，如 `bash`、`python`、`markdown`，便于高亮与复制。
- **块内注释**：可加简短注释说明参数或步骤，注释与内容之间留空格。

---

## 排版与风格

- **段落**：列表项与正文之间、代码块前后各留一空行；列表项内若多段，用 2 空格缩进区分层级。
- **术语**：全文保持同一套术语（如统一用「环境搭建」或「安装」其一）。
- **简洁**：避免冗长说明，优先用列表和代码块表达步骤；复杂说明可单独放到文档或链接。

---

## 可选内容

- **目录**：长 README 可在标题下用 `- [章节名](#锚点)` 列出目录。
- **徽章/链接**：若有仓库、文档、版本号等，可放在顶部简介下方。
- **贡献/许可**：若项目需要，可在文末增加「贡献说明」「许可证」等小节。

---

标题清晰、章节编号、代码块带语言、列表与空行统一，术语一致，整体简洁易扫读。
