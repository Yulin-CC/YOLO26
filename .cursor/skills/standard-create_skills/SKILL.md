
# 📜 如何新建一个 Skill

  - **概述**: 当**创建一个新的 SKILL 时**（`.md`）时，请按照下面的一些规范进行文档编写的约束，且注意保持简洁美观。
  - **日期**: 2026-03-13

## 1 文件结构

  - 新建的 SKILL 放在当前项目中的 `.cursor/skills/` 目录下
  - 其命名范式为`standard-功能/SKILL.md`：
  - 示例如下

    ```markdown
    ├──.cursor/skills/
    │ ├── standard-xxx
    │ │   └── SKILL.md
    │ └── standard-xxx
    │     └── SKILL.md
    ```

## 2 内容规范

### 2.1 Header

  - SKILL 标题：emoji + 名称
  - 标题下方附文，说明本 Skill 的触发场景与创建时间。字段含义：
    - **概述**：一句话说明本 Skill 的适用场景（何时触发、做什么）。
    - **日期**：使用**当前日期**，格式 `YYYY-MM-DD`。

  - 示例如下：

    ```markdown
    # 📜 Skill 标题
    
      - **概述**: 简述本 skill 的功能与触发时机。
      - **日期**: 2026-03-13
    ```

### 2.2 正文结构

  - **标题格式**：
    - 一级标题：（`## 1`、`## 2`、...）。
    - 二级标题：（`### 2.1`、`### 2.2`、...）。
    - 三级标题：（`#### 2.1.1`、`#### 2.1.2`、...）
    - 四级标题：（`**(1)**`、`**(2)**`、`**(3)**`、...）
    - 五级标题：（`**a**`、`**b**`、`**c**`、...）
    - 六级标题：（`**(a)**`、`**(b)**`、`**(c)**`、...）

  - 每个小节应包含：**说明文字** + **示例代码块**（如适用）+ **注意事项**（如有）。

  - 示例结构如下：

    ```markdown
    ## 1 第一节标题

      - 说明文字……

        ```示例代码或伪代码```

      - **注意**：补充约束或边界条件。
    ```

  - **注意**：各节之间保持逻辑递进，避免内容重叠；示例代码务必能直接参考或运行。

### 2.3 书写风格

  - **语言**：默认使用中文，专有名词（函数名、字段名、命令等）保留英文原文。
  - **列表缩进**：正文列表项统一使用 **2 个空格**缩进，嵌套层再加 2 个空格。
  - **强调**：关键词、字段名、路径使用 `**粗体**` 或 `` `行内代码` `` 标注，避免过度强调。
  - **代码块**：必须标注语言（如 ` ```python `、` ```markdown `、` ```bash `）以启用语法高亮。
  - **简洁原则**：每条规范用一句话表达核心约束，避免冗余描述。

  - 示例（正确风格）：

    ```markdown

    - SKILL 标题：emoji + 名称

    - 标题下方附文，说明本 Skill 的触发场景与创建时间。字段含义：

      - **概述**：一句话说明本 Skill 的适用场景（何时触发、做什么）。
      - **日期**：使用**当前日期**，格式 `YYYY-MM-DD`。
      
    ```
